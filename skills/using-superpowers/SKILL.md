# Using Superpowers (oh-my-grok)

If `vendor/superpowers/skills` is present, prefer those skills. Otherwise use this bootstrap.

## Core flow

1. **brainstorming** — clarify intent, design in sections, get approval
2. **writing-plans** — bite-sized tasks with verification
3. **test-driven-development** — red → green → refactor
4. **systematic-debugging** — when things break
5. **verification-before-completion** — evidence before "done"
6. **requesting-code-review** / finish branch skills when applicable

## Rule

Do not jump into coding on ambiguous requests. Read the matching skill first (Skill Gate will enforce Read).

Vendor sync:

```bash
npm run vendor:superpowers
```
